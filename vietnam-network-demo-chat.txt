## Kiến trúc tổng quan

**Backend:** Graph database + thuật toán BFS (Breadth-First Search)
**Frontend:** Web app hiển thị chuỗi kết nối

---

## Cần chuẩn bị gì

### 1. Công nghệ core
| Phần | Lựa chọn đề xuất | Lý do |
|------|-----------------|-------|
| Database | **Neo4j** (graph DB) | Được thiết kế cho bài toán quan hệ, query nhanh |
| Backend | **Node.js + Express** hoặc Python FastAPI | Dễ kết nối Neo4j |
| Frontend | **React** | Có thể dùng thư viện vẽ graph như `vis.js` |

### 2. Dữ liệu
- Cần có cách để **người dùng tự đăng ký** và **kết bạn/xác nhận quen nhau**
- Hoặc nếu là hệ thống nội bộ: import danh sách quan hệ thủ công

### 3. Thuật toán
- **BFS** (tìm đường ngắn nhất) — Neo4j có sẵn query `SHORTEST PATH`
- Ví dụ query Neo4j:
  ```cypher
  MATCH path = shortestPath((a:Person {name:"A"})-[:KNOWS*]-(d:Person {name:"D"}))
  RETURN path
  ```

## Các tính năng cơ bản cần làm

1. **Đăng ký / đăng nhập** người dùng
2. **Thêm kết nối** (A quen B — cả hai xác nhận)
3. **Tìm kiếm** nhập tên người muốn kết nối → hiển thị chuỗi A→B→C→D
4. **Visualize** đồ thị quan hệ (optional nhưng đẹp)

- **Hệ thống đầy đủ** (đăng ký, xác nhận kết nối, visualize): ~1-2 tháng
